  <action>
Create `src/engine/timelineGates.ts` as a pure engine module (no React imports). Follow the project engine module convention with the banner comment header.

Export these items:

1. `IRL_GATE_DURATIONS: IrlGateDurations` — named constants for seed_check (1-2), due_diligence (1-3), series_a_check (2-4), lp_close (6-12), follow_on (1-2), board_meeting (1-1). All values are in-game months (1 advanceTime() = 1 month).
2. `checkTimeGate(fund: Fund, actionType: string)` — returns `{ blocked: boolean; availableFromMonth?: number; monthsRemaining?: number }`. 3. `openTimeGate(fund: Fund, actionType: string, reason: string)` — creates a new `TimeGate` with `availableFromMonth = fund.currentMonth + randomDelay`. Delay is random between `IRL_GATE_DURATIONS[actionType].min` and `.max` (rounded). Falls back to delay of 1 if actionType not in durations.
4. `clearExpiredGates(fund: Fund)` — returns filtered `TimeGate[]` keeping only gates where `availableFromMonth > fund.currentMonth`.

Import types from `./types` using `import type`. Use `randomBetween` from `@/lib/utils` for delay calculation (it already exists there).

Then create `src/engine/timelineGates.test.ts` with tests covering all behaviors listed above. Use `describe`/`it` blocks. Create mock Fund objects inline with only the fields needed (timelineMode, activeTimeGates, currentMonth). For openTimeGate tests, accept any value in the valid range rather than testing randomness exactly.
