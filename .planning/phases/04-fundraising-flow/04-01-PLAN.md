<feature>
  <name>Fundraising Engine Module</name>
  <files>src/engine/fundraising.ts, src/engine/fundraising.test.ts</files>
  <behavior>
    - generateLPProspects(fundNumber, fundSize, lpSentimentScore): returns 6-12 LPProspect objects with randomized names, types, targetCommitment (5-20% of fund size), interestLevel (30-90 scaled by sentiment), relationshipScore (20-80), all starting with status "prospect"
    - calculatePitchOutcome(prospect, lpSentimentScore, marketCycle): advances status by one step (prospect->pitched->soft_circle->hard_commit->closed); probability = base 0.6 * (interestLevel/100) * (relationshipScore/100) * commitmentMod(lpSentimentScore); returns {newStatus, message}; declined prospect returns {newStatus: "declined", message: "Already declined"}
    - calculateTotalCommitted(prospects): sums targetCommitment for prospects with status "soft_circle", "hard_commit", or "closed"
    - canStartNextFund(fund): returns true when fund.tvpiEstimate >= fund.nextFundUnlockTvpi AND fund.fundNumber < 3; returns false otherwise
    - getFirstCloseThreshold(targetAmount): returns targetAmount * 0.5
    - getFinalCloseThreshold(targetAmount): returns targetAmount
    - calculateNegotiatedFundSize(prospects, marketCycle, lpSentimentScore): returns total committed * marketMod (bull=1.1, bear=0.85, neutral=1.0) * sentimentMod (lpSentimentScore / 50, clamped 0.7-1.3)
    - FUND_II_TVPI_THRESHOLD = 2.0, FUND_III_TVPI_THRESHOLD = 2.5
    - DEFAULT_FUND_TERMS = { managementFee: 0.02, carry: 0.20, hurdleRate: 0.08, fundLife: 10, gpCommitPercent: 0.01 }
  </behavior>
  <implementation>
    Create src/engine/fundraising.ts as a pure module (no React imports) following the same pattern as timelineGates.ts and lpSentiment.ts. All functions are exported and stateless — they receive data, return results, and never mutate inputs.

    Key design decisions:
    - LP prospect generation uses deterministic-feeling randomness (seeded by fundNumber + fundSize) — same inputs produce varied but bounded outputs
    - Pitch probability formula: baseProbability * (interestLevel / 100) * (relationshipScore / 100) * commitmentMod. Base probability is 0.6 for prospect->pitched, 0.5 for pitched->soft_circle, 0.4 for soft_circle->hard_commit, 0.7 for hard_commit->closed
    - commitmentMod derived from lpSentimentScore: (lpSentimentScore / 50) clamped to [0.7, 1.3] — same concept as getLPEffects().commitmentMod
    - Market cycle multiplier for fund size: bull_market=1.1, bear_market=0.85, recovery/downturn/stable=1.0
    - canStartNextFund also requires fundNumber < 3 (max 3 funds)
    - All user-facing strings use t() from @/lib/i18n

    Anti-patterns to avoid:
    - Do NOT build LP archetype behavior (FUND-V2-01, deferred)
    - Do NOT import React or Zustand — pure functions only
    - Do NOT use magic numbers — all thresholds as named constants

  </implementation>
